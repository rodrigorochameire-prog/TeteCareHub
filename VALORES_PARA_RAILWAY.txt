📋 VALORES PARA PREENCHER NO RAILWAY
═══════════════════════════════════════════════════════════════

Após o agente criar as variáveis, preencha com estes valores:

───────────────────────────────────────────────────────────────

1. DATABASE_URL
   ───────────────────────────────────────────────────────────
   Como obter:
   1. Vá em: Supabase Dashboard → Settings → Database
   2. Role até "Connection string"
   3. Selecione "URI" (NÃO "Session mode" ou "Transaction mode")
   4. Copie a string completa
   
   Exemplo:
   postgresql://postgres.abcdefghijklmnop:[SENHA]@aws-0-us-east-1.pooler.supabase.com:5432/postgres
   
   OU conexão direta:
   postgresql://postgres:[SENHA]@db.abcdefghijklmnop.supabase.co:5432/postgres

───────────────────────────────────────────────────────────────

2. SUPABASE_URL
   ───────────────────────────────────────────────────────────
   Como obter:
   1. Vá em: Supabase Dashboard → Settings → API
   2. Copie o "Project URL"
   
   Exemplo:
   https://abcdefghijklmnop.supabase.co

───────────────────────────────────────────────────────────────

3. SUPABASE_ANON_KEY
   ───────────────────────────────────────────────────────────
   Como obter:
   1. Vá em: Supabase Dashboard → Settings → API
   2. Em "Project API keys"
   3. Copie a chave "anon" (public) - é a chave que começa com "eyJ..."
   
   Exemplo:
   eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoiYW5vbiIsImlhdCI6MTY0NTk4ODgwMCwiZXhwIjoxOTYxNTY0ODAwfQ.xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

───────────────────────────────────────────────────────────────

4. SUPABASE_SERVICE_ROLE_KEY
   ───────────────────────────────────────────────────────────
   Como obter:
   1. Vá em: Supabase Dashboard → Settings → API
   2. Em "Project API keys"
   3. Copie a chave "service_role" (secret) - é a chave que começa com "eyJ..."
   ⚠️ CUIDADO: Esta chave tem acesso TOTAL ao banco! NUNCA exponha no frontend!
   
   Exemplo:
   eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoic2VydmljZV9yb2xlIiwiaWF0IjoxNjQ1OTg4ODAwLCJleHAiOjE5NjE1NjQ4MDB9.yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy

───────────────────────────────────────────────────────────────

5. NODE_ENV
   ───────────────────────────────────────────────────────────
   Valor fixo:
   production

───────────────────────────────────────────────────────────────

6. VITE_SUPABASE_URL
   ───────────────────────────────────────────────────────────
   Valor:
   Mesmo valor do SUPABASE_URL
   (Exemplo: https://abcdefghijklmnop.supabase.co)

───────────────────────────────────────────────────────────────

7. VITE_SUPABASE_ANON_KEY
   ───────────────────────────────────────────────────────────
   Valor:
   Mesmo valor do SUPABASE_ANON_KEY
   (Exemplo: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...)

───────────────────────────────────────────────────────────────

VARIÁVEIS OPCIONAIS (se usar Stripe):

8. STRIPE_SECRET_KEY
   Como obter: Stripe Dashboard → Developers → API keys → Secret key

9. STRIPE_WEBHOOK_SECRET
   Como obter: Stripe Dashboard → Developers → Webhooks → Add endpoint → Copy signing secret

10. VITE_STRIPE_PUBLISHABLE_KEY
    Como obter: Stripe Dashboard → Developers → API keys → Publishable key

───────────────────────────────────────────────────────────────

VARIÁVEL OPCIONAL (JWT Secret):

11. JWT_SECRET
    Gerar comando:
    openssl rand -hex 32
    
    Ou use qualquer string aleatória segura de pelo menos 32 caracteres

───────────────────────────────────────────────────────────────
